NARRATOR: Chapter twenty, A-I versus machine learning. A-I is the broad field of building systems that behave intelligently — reason, perceive, decide — and they may or may not learn. Machine learning is a subset of A-I that learns patterns from data and improves with experience — every machine learning system is A-I, but not the other way around. The dividing line is learning: rules a human wrote versus patterns learned from data. A rule-based expert system has hand-written rules, is transparent, and doesn't learn; a machine learning classifier learns its labels from data. Then the three automation approaches. DevOps automates the build-test-deploy pipeline — C-I-C-D, inward at the team. R-P-A is task-level — it mimics a human at the user interface, like copy-pasting invoices; low-code but brittle to interface change. B-P-A is process-level — it orchestrates an end-to-end workflow with deep system integration, human approvals, and an audit trail, like onboarding. Mnemonic: R-P-A is a Robot doing a Task; B-P-A is redesigning a Process. And machine learning augments automation by adding judgement at decision points — "automation provides the hands, machine learning provides the eyes and judgement." The trap: A-I is not machine learning — machine learning is a subset; and a plain if-statement is not A-I.

NARRATOR: Chapter twenty, the models of training machine learning — each defined by the data it needs. Mnemonic S-U-S-R: Supervised, Unsupervised, Semi-supervised, Reinforcement. Supervised uses labelled data — input plus known correct output — like an answer key; it splits into classification, predicting a category, and regression, predicting a continuous number; a spam filter is the example — and it's only as good as its labels, which is where human and historical bias enters. Unsupervised has no labels and discovers hidden structure itself — clustering for customer segmentation, anomaly detection. Semi-supervised uses a few expensive labels plus a large unlabelled pool, because labelling is costly — medical imaging is the classic. Reinforcement has no fixed dataset — an agent acts in an environment and learns by trial and error from a reward or penalty signal, like games or robotics. The decision: do you have labels, how many, or is it an environment with rewards? The trap: supervised versus unsupervised is purely "do you have labels?"; and reinforcement is not supervised, because a reward after acting tells you how good, never what the right action was.

NARRATOR: Chapter twenty, common applications of machine learning. Mnemonic F-A-I, said "fie": Forecasting, Assistants, Image recognition. Forecasting and data analysis find patterns and anomalies and predict future numeric values — the technique underneath is supervised regression on labelled history; it assumes the future resembles the past, so it fails when the world shifts — Zillow Offers over-bought houses and lost around five hundred million. Virtual personal assistants understand and respond to human language — supervised learning plus natural language processing, trained on vast language data. Image recognition interprets and classifies visual content — faces, scans, self-driving sight — using deep neural networks, usually supervised on large labelled image sets, semi-supervised where labels are costly. The answer shape: name the application, what it does, the technique underneath, and the data it needs — the technique link is the mark students miss. The trap: treating machine learning as magic — always name the data requirement and the failure mode.

NARRATOR: Chapter twenty, the models to design and analyse machine learning — decision trees and neural networks. A decision tree predicts by asking a sequence of if-then test questions, following branches to a final prediction — the same selection logic as Year eleven, but the questions and their order are learned from data. The parts, mnemonic R-N-L: Root — the first question at the top; Node — a further test on a feature; Leaf — the end-point prediction. To trace it, follow root to leaf and state the path, not just the leaf. A tree is interpretable — you can explain exactly why. A neural network is many simple neurons in layers — input, hidden, output — joined by weighted connections; a neuron is a weighted sum of inputs then an activation function, and learning means adjusting the weights; "layers learn features" — edges then faces. A network is a black box — the decision is spread across many weights with no human-readable rule. The key comparison, and the exam question, is explainability: a tree is interpretable but less powerful on complex tasks; a network is powerful but opaque — a trade-off of transparency versus power, and explainability ties to accountability, the last A of CIA-AAA. Mnemonic: a Tree is readable if-questions; a Net is weighted neurons. The trap: a neural network is not a brain — just layers of weighted sums and activations; and "deeper is better" is wrong, because too deep overfits.

NARRATOR: Chapter twenty, R-P-A and B-P-A practical considerations. The decision: automate the task with R-P-A, or rethink the process with B-P-A? R-P-A is a quick, low-code, tactical patch that treats the symptom; B-P-A is an integrated, redesigned, strategic fix that treats the cause. Implementation: R-P-A sits on low-code visual designers, on top of existing user interfaces with no system change — good for legacy — and screen-scrapes; B-P-A uses deep integration with proper interfaces, a workflow engine, human approvals, and an audit trail. The central R-P-A risk: it's brittle — it breaks when the user interface changes, because it mimics a human at the screen rather than talking to a stable A-P-I, so it's an ongoing maintenance burden, not a permanent fix. "R-P-A is brittle — it breaks when the user interface changes." Security is Secure Software Architecture applied to automation: protect the bot's credentials — never hard-code keys, use a secrets manager — apply least privilege, validate inputs, and respect privacy. Ethics, where the verb is evaluate: employment is a shift in work plus reskilling, not just jobs gone; plus accountability for errors and the audit trail. The evaluate shape: benefits, then costs and risks, then a justified judgement.

NARRATOR: Chapter twenty-one, regression and the core algorithm types — where the programming begins. Linear regression predicts a continuous number by fitting the best straight line — output is slope times x plus intercept, the learned parameters. Logistic regression, despite the name, is classification — it predicts a category, yes or no, via the sigmoid, which squashes any number to between zero and one as a probability, with a threshold at point-five; that sigmoid is the neuron's activation. Polynomial regression fits a curve where a line can't — and it's how you overfit. K-nearest neighbour predicts for a new point by finding the K most similar training examples and taking a majority vote or average — no line or curve, just comparison. The core algorithm types, mnemonic L-L-K: Linear regression, Logistic regression, K-nearest neighbour — and the marquee distinction, "Linear predicts a number; Logistic predicts a yes-or-no." In object-oriented terms, models are fit-and-predict classes — fit learns the parameters from data, predict applies them, exactly like scikit-learn's dot-fit and dot-predict. Overfitting is when a model fits the training data too closely, memorising noise, so it's great on training and fails on new data; reduce it with a simpler model, more data, or regularisation. And the train-test split: never evaluate on training data — train on one set, test on a separate unseen set; a big gap means overfitting. The trap: logistic "regression" is classification, not number-prediction — output type decides; and a perfect training fit is a red flag, not a triumph.

NARRATOR: Chapter twenty-one, neural networks — concepts and implementation. The verb is apply: describe, trace, and explain, not derive calculus. A perceptron is the simplest network — a single neuron: inputs times weights, summed with a bias to give a weighted sum, through an activation to an output; weights are learned, the bias is an adjustable threshold — roughly logistic regression, one neuron with a sigmoid. The forward pass runs inputs through to a prediction — multiply, sum, activate — and that's what "trace" questions test. The structure, for the "describe the structure" question: an input layer with one neuron per feature, one or more hidden layers each doing weighted-sum-then-activation and passing forward, and an output layer, all joined by weighted links — "layers learn features," early layers edges, later layers faces. Training is the P-L-A loop: Predict — the forward pass; Loss — how far from the known label; Adjust — nudge every weight to reduce the error — repeated over many epochs. Technically that's backpropagation, pushing the error back to find each weight's contribution, plus gradient descent, stepping downhill — name them for polish, but the loop is the substance, and it's the same optimisation as fitting regression, scaled up. The trap: hand-waving "it just learns the patterns" is a band-four answer — you must give the predict-loss-adjust loop; and "bigger is better" is wrong, because more capacity means more overfitting.

NARRATOR: Chapter twenty-two, assessing the impact of automation. The verb is assess or evaluate — weigh both sides and reach a judgement, not a list of bad things. The five named impacts sit in four buckets, mnemonic S-P-E-E: Safety and access — workers plus people with disability; People and skills — employment; Efficiency and Environment; and Economy and wealth. Safety: removes humans from hazardous work, but creates new hazards and deskilling. Disability: enabling when designed inclusively — voice assistants, captions — but excluding when not. Employment, the most nuance: not jobs taken but a transformation — it displaces routine jobs, creates new build-and-maintain roles, but with a skills gap and reskilling burden. Efficiency and environment: more output and less production waste, but a hardware footprint, e-waste, and large model training using huge electricity and water — A-I's own carbon cost. Economy and wealth: grows the economy and cheapens goods, but the gains concentrate with capital owners while displaced workers pay — inequality, depending on policy. The answer shape: upside, downside, each with an example, then a judgement. The trap: one-sided answers — assess means both sides plus a judgement; "robots take jobs" with no skills-shift nuance; and forgetting A-I's own environmental cost.

NARRATOR: Chapter twenty-two, human behaviour patterns and their influence on machine learning and A-I. The core flip: the previous lesson was automation's impact on humans; this is human behaviour's influence on the system — its data and its acceptance — because these systems learn from human-generated data and depend on human acceptance. The four named factors, mnemonic P-S-C-B — "Please See, Curious Brain": Psychological responses, acute Stress response, Cultural protocols, Belief systems. And "by implementation" means name the factor and the concrete design response — the mark students miss. Psychological: trust needs transparency and explainability, people want override, and too much automation raises cognitive load — so explain decisions and keep human override. Acute stress response: fight-or-flight makes behaviour less accurate, and crucially a model trained only on calm data fails the stressed user when they need it most — the three-a-m clinician — so train on realistic high-stress data. Cultural protocols: authority, communication style, individual versus collective decision-making, and privacy expectations all vary — including indigenous cultural and intellectual property for Aboriginal and Torres Strait Islander material — so localise and test with target cultures using diverse teams. Belief systems: tech-optimism, human-agency, privacy-as-fundamental decide whether people accept or reject the same feature — so offer transparency, oversight, and a choice of automation level. The thesis: successful A-I adapts to human behaviour; it doesn't force humans to adapt. The trap: treating all users as uniform — the central error; and a vague "be careful with feelings" with no named factor and no design response.

NARRATOR: Chapter twenty-two, bias in datasets and models — the last teaching lesson. Bias is a systematic error producing unfair or skewed outcomes for particular groups — not random noise. The myth to kill: "the algorithm is neutral and objective" — wrong; the maths is impartial, but it faithfully learns the data's patterns, so it mirrors and amplifies the biases in human-generated data, even when the sensitive attribute isn't an input. The four sources, mnemonic S-H-L-M, said "SHaLuM" — a bad sample of the world: Sampling — some groups missing or under-represented, like face recognition trained on light-skinned faces; the tell is "who is missing?" Historical — the data is a faithful record of an unjust past, so the model perpetuates discrimination, like a hiring model on male-dominated history; the tell is "a true record of an unfair past?" and more data does not fix it. Labeler — humans' bias baked into the labels in supervised learning; the defence is guidelines and multiple labelers. Measurement — a flawed proxy distorts the data, like arrests as a proxy for crime in over-policed areas. The mitigations, mnemonic D-R-F — "Don't Reinforce unFairness": Diverse data at the source, Reweight in training with fairness constraints, and Fairness metrics afterward, measured per group, not overall — because high overall accuracy can still be unfair to a minority. The accountability lift: model cards disclosing training data and per-group performance, plus reproducibility and data provenance — that's the accountability A of CIA-AAA applied to A-I. The trap: "the algorithm is neutral"; "more data fixes bias" — if the history is biased, more is more bias; and checking only overall accuracy.

QUESTION: Question one. Distinguish between artificial intelligence and machine learning, and give an example of an A-I system that is not machine learning. Pause, attempt it, then play.

NARRATOR: Model answer: artificial intelligence is the broad field of building systems that behave intelligently — reasoning, perceiving, deciding — and they may or may not learn. Machine learning is a subset of A-I in which the system learns patterns from data and improves with experience. The dividing line is learning. An A-I system that is not machine learning is a rule-based expert system, where a human writes the rules explicitly — it behaves intelligently and is transparent, but never learns from data. "A-I is the goal, machine learning is one way." Stating that machine learning is a subset, that the line is learning, and giving the rule-based example is the mark.

QUESTION: Question two. A bank trains a loan-approval model on twenty years of its own lending decisions. Explain why this model may be biased and how that bias could be reduced. Pause, then play.

NARRATOR: Model answer: the model may be biased because of historical bias — the training data is a faithful record of past lending decisions, so if the bank historically approved some groups less often, the model learns and perpetuates that discrimination, even if the protected attribute isn't an input, because it finds correlated proxies; the algorithm isn't neutral, it mirrors its data, and crucially more of the same historical data only deepens the bias. To reduce it, use D-R-F: source diverse, representative, contemporary data; reweight or apply fairness constraints in training; and measure fairness metrics per group after training — not just overall accuracy — plus accountability through model cards and human oversight. Naming historical bias, the not-neutral mechanism, and a real mitigation is the full-mark answer.

QUESTION: Question three, the harder one. A hospital emergency department wants an A-I tool to triage patients by urgency. Assess two impacts of deploying it and recommend two responsible-A-I safeguards. Pause the player, plan an answer, then play.

NARRATOR: Model answer: assess, so both sides plus a judgement. Impact one, from S-P-E-E People-and-skills and Safety: it can speed triage and remove routine load, improving safety when correct, but it risks deskilling clinicians and over-reliance, and a wrong urgency call is high-stakes — so it should augment, not replace, clinical judgement. Impact two, efficiency versus fairness: it improves throughput, but if trained on biased data — S-H-L-M, perhaps sampling or measurement bias — it could systematically under-triage a particular group, a serious equity harm. Safeguards: first, human-in-the-loop — the tool is decision support, a clinician confirms every urgent-or-not call, which is also accountability, the last A of CIA-AAA; second, a bias audit — measure fairness metrics per group, train on representative high-stress data so it works for stressed and atypical patients, P-S-C-B, and publish a model card. A full-mark answer assesses two impacts two-sidedly, names a bias source, and gives concrete safeguards — human oversight plus a per-group bias check.

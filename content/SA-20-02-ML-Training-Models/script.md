NARRATOR: Now today. Machine learning learns patterns from data — but how does it actually learn? There isn't one way; there are several, and which one you use depends entirely on what data you have. Today's dot-point, in the syllabus's words: explore models of training machine learning — supervised learning, unsupervised learning, semi-supervised learning, and reinforcement learning. The outcome is still S-E-twelve-oh-three. These four are the training paradigms, and "distinguish between" any pair of them, or "identify the most appropriate model for this scenario and justify", are guaranteed exam questions. So today's whole job: know what data each one needs, and what each one is for.

NARRATOR: Objectives up front. By the end you'll define all four training models; for any scenario, pick the right one and justify it by pointing at the data; and avoid the two traps markers exploit — confusing supervised with unsupervised, and thinking reinforcement learning is just supervised learning. Those distinctions are where the marks live.

NARRATOR: Here is the memory hook for the four, and it's the spine of the episode. The four training models are Supervised, Unsupervised, Semi-supervised, and Reinforcement. Take the first letters: S, U, S, R. Remember it as S-U-S-R — say it "suss-er", as in you're trying to suss out how the machine learns. Write S-U-S-R in the margin and you've got all four. But the letters alone aren't enough — each one needs a trigger word, the one-word cue for what data it uses. Supervised is labelled. Unsupervised is find-structure. Semi-supervised is few-labels. Reinforcement is reward. So: S-U-S-R, and labelled, find-structure, few-labels, reward. Lock those four trigger words in, because the trigger word is what tells you which model a scenario needs.

NARRATOR: First model, supervised learning. The trigger word is labelled. Supervised learning trains a model on a labelled dataset — every example comes with both the input and the correct answer. You show the system thousands of examples where you already know the right output, it learns the mapping from input to output, and it can then predict the answer for new, unseen examples. Think of it as learning with an answer key: you "supervise" the learner by giving it the correct answers during training. The classic example is an email spam filter. You feed it thousands of emails each already tagged "spam" or "not spam" — those tags are the labels — and it learns which patterns predict spam, so it can label a brand-new email. Other examples: medical diagnosis where past cases have a known diagnosis, or price prediction where past sales have a known price. Supervised learning splits into two jobs: classification, where the answer is a category like spam-or-not, and regression, where the answer is a continuous number like a price — and regression is the whole topic of a later episode. The supervised fit-and-predict shape is in Listing one, and the labelled-versus-unlabelled data shapes are in Listing two. One line for the exam: supervised learning learns from labelled data — inputs paired with known correct outputs — to predict the output for new inputs.

NARRATOR: Here's where last episode's thread pays off. Supervised learning is only as good as its labels — and the labels come from humans. If the humans who labelled the data were biased, or the historical data was biased, the model learns that bias as if it were truth. That is "garbage labels in, bias out", and it's not hypothetical. A story to anchor it. Amazon built an experimental machine learning tool to screen job applicants' résumés. It was supervised — trained on ten years of past hiring decisions, with the outcomes as labels. The problem: past hiring had favoured men, so the model learned that pattern and started penalising résumés containing the word "women's", as in "women's chess club captain". The labels encoded a historical bias and the model faithfully reproduced it. Amazon scrapped it. The lesson, which you can write in any bias question: supervised learning learns whatever its labels teach it, including human prejudice — the model wasn't malfunctioning, it was working exactly as trained on biased data.

NARRATOR: Second model, unsupervised learning. The trigger word is find-structure. Unsupervised learning works on data with no labels — just the inputs, no correct answers. There's no answer key, so the system can't be told what's right; instead it discovers hidden structure on its own — groupings, patterns, relationships nobody told it to look for. The classic example is customer segmentation: you give it data on thousands of customers — age, income, spending, online activity — with no categories attached, and it finds natural clusters, like "young high-spenders" and "older conservative buyers", that you didn't define in advance. The technique there is clustering. Unsupervised learning is also how a lot of anomaly detection works — it learns what "normal" looks like and flags what doesn't fit. The exam contrast is direct: supervised has labels and predicts a known answer; unsupervised has no labels and discovers unknown structure. One line: unsupervised learning finds hidden patterns or groupings in unlabelled data, without being given correct answers.

QUESTION: This is the distinction markers test most, so pause on it. You have a dataset of ten thousand customers and you want to group them into similar segments — but you have no pre-existing categories. Supervised or unsupervised, and why? Pause the player, decide, then play.

NARRATOR: Unsupervised, because there are no labels. The giveaway is "no pre-existing categories" — you don't have a correct answer for each customer to learn from, so you can't supervise. You're asking the system to find the structure itself, which is clustering, which is unsupervised. If the scenario had instead said "we have customers already tagged as high-value or low-value and we want to predict which a new customer will be", that flips to supervised, because now there are labels to learn from. The whole decision hinges on one question: do you have labels? Labels means supervised; no labels and you want groups or patterns means unsupervised. That single question answers most of these.

NARRATOR: Third model, semi-supervised learning. The trigger word is few-labels. Semi-supervised learning is the realistic middle ground: a small amount of labelled data and a large amount of unlabelled data, used together. Why does it exist? Because labelling is expensive. Getting a human expert to label data — a radiologist labelling medical scans, say — costs time and money, so you can usually only afford to label a little. But unlabelled data is cheap and plentiful. Semi-supervised learning uses the few labels you can afford to anchor the learning, then leverages the large pool of unlabelled data to learn the broader structure, getting better results than the few labels alone could give. The classic example is exactly that medical imaging case: a thousand scans labelled by experts, fifty thousand unlabelled, and you use all fifty-one thousand. One line: semi-supervised learning combines a small labelled dataset with a large unlabelled one, to get strong results when labelling is too expensive to do at scale. The shape of a partly-labelled dataset is in Listing three.

NARRATOR: Fourth model, reinforcement learning. The trigger word is reward. Reinforcement learning is genuinely different from the other three — there's no fixed dataset at all. An agent interacts with an environment, takes actions, and learns from the rewards or penalties it gets back. It learns by trial and error: try an action, see whether the outcome was good or bad, adjust to favour actions that lead to reward over time. Think of training a dog with treats, or learning a video game by playing it — you don't get told the right move, you get a score, and you figure out which moves raise it. The classic examples are game-playing systems that learned chess or Go by playing millions of games against themselves, robotics learning to walk, and self-driving navigation. The defining features: an agent, an environment, actions, and a reward signal — plus actions are sequential, and a reward might come long after the action that earned it. One line: reinforcement learning trains an agent to make a sequence of decisions by trial and error, learning from rewards and penalties rather than from a labelled dataset.

NARRATOR: Now the trap on reinforcement learning, because it's a favourite. Students see "it learns from feedback" and think that's just supervised learning — the reward is the label, right? No, and you lose the mark if you write that. In supervised learning you're given the correct answer for every example up front, before any learning — a fixed answer key. In reinforcement learning there is no correct-answer key; the agent only gets a reward signal after it acts, telling it how good the outcome was, not what the right action would have been. Supervised says "the answer was spam"; reinforcement says "that move lost you three points" — it never says what the right move was. The agent has to explore and discover good actions itself. So: supervised learns from labelled answers; reinforcement learns from rewarded consequences. Different feedback, different mechanism. Don't blur them.

NARRATOR: Now the decision framework, because the highest-value question is "which model for this scenario, and justify". The decision tree is simple — run it in your head. Do you have labelled data? If yes and lots of it, and you want to predict a known answer — supervised. No labels and you want to discover groups or patterns — unsupervised. A few labels, lots of unlabelled data, and labelling more is expensive — semi-supervised. No fixed dataset but an environment that gives feedback for actions over time — reinforcement. So the questions are: labels? how many? or is it an environment with rewards? Let me drill it with four real problems.

NARRATOR: Worked example, four scenarios, classify each into S-U-S-R and justify. Scenario one: an email spam filter, with millions of emails already tagged spam or not-spam. Supervised — abundant labels, predicting a known category. Scenario two: a retailer wants to discover natural groupings among its customers, with no pre-existing segments. Unsupervised — no labels, finding structure. Scenario three: a fraud-detection system where only a handful of transactions have been confirmed as fraud by investigators, but there are millions of unlabelled transactions. Semi-supervised — a few expensive labels, a huge unlabelled pool. Scenario four: a game-playing A-I that should learn to win, able to play and get a score. Reinforcement — an environment with a reward signal and sequential decisions. Notice the justification each time points straight at the data: "because you have labels", "because you have none", "because labels are few and expensive", "because it's an environment with rewards". That's the mark — never just name the model; justify it by the data.

NARRATOR: Now the weak-versus-strong contrast on a justify question. Scenario: "a hospital has a thousand expert-labelled X-rays and fifty thousand unlabelled ones; recommend a training model." A weak answer: "Use supervised learning because it's accurate." That ignores the data situation and the cost of labelling — it throws away fifty thousand images and pretends labelling is free. A strong answer: "Semi-supervised learning is most appropriate, because expert labelling of X-rays is expensive and the hospital has only a small labelled set, but a large pool of unlabelled scans. Semi-supervised learning uses the thousand labelled images to anchor the model and the fifty thousand unlabelled images to learn the broader structure, achieving better accuracy than the labelled data alone could, without the prohibitive cost of labelling everything." Hear the difference — the strong answer names the model, justifies it by the specific data situation and the cost of labels, and states the effect. Data situation, model, effect.

NARRATOR: Lock in today. Four training models — S-U-S-R, suss-er. Supervised is labelled — learn from input-output pairs to predict a known answer. Unsupervised is find-structure — discover groupings in unlabelled data. Semi-supervised is few-labels — combine a small labelled set with a large unlabelled one when labelling is expensive. Reinforcement is reward — an agent learns by trial and error from rewards in an environment, no labelled dataset. To pick one, ask: labels? how many? or an environment with rewards? And always justify by the data. The traps: supervised versus unsupervised is all about whether you have labels; and reinforcement is not supervised — reward after acting is not a labelled answer key.

QUESTION: Question one, recall. Name the four models of training machine learning, and give the one-word data cue for each. Pause, then play.

NARRATOR: Model answer: supervised learning — labelled data; unsupervised learning — unlabelled data, find structure; semi-supervised learning — a few labels plus a large unlabelled set; reinforcement learning — learning from rewards in an environment. The hook is S-U-S-R. Naming all four correctly with the right data cue is the mark.

QUESTION: Question two, the classic distinction. Distinguish between supervised and unsupervised learning. Four marks. Pause, then play.

NARRATOR: Model answer: supervised learning trains on a labelled dataset, where each input is paired with a known correct output, and learns to predict that output for new inputs — for example, classifying email as spam or not-spam from labelled examples. Unsupervised learning trains on unlabelled data, with no correct answers provided, and instead discovers hidden structure such as natural groupings — for example, clustering customers into segments that were not defined in advance. The key distinction is the presence of labels: supervised uses labelled data to predict a known answer, while unsupervised uses unlabelled data to discover unknown structure. Stating both definitions, an example each, and the label-based contrast earns full marks.

QUESTION: Question three. Explain how reinforcement learning differs from supervised learning. Pause, then play.

NARRATOR: Model answer: in supervised learning the model is given a fixed dataset in which every example has the correct answer as a label, and it learns the mapping from input to output before being used. In reinforcement learning there is no labelled dataset; an agent interacts with an environment, takes actions, and receives a reward or penalty indicating how good the outcome was — but it is never told the correct action. The agent learns by trial and error over a sequence of decisions to maximise total reward. So supervised learning learns from labelled correct answers, while reinforcement learning learns from rewarded consequences of its own actions. Making clear that a reward signal is not a correct-answer label is the mark.

QUESTION: Question four, application. A streaming company has years of viewing data but no labels, and wants to discover groups of similar viewers to inform recommendations. Identify the most appropriate training model and justify your choice. Pause, then play.

NARRATOR: Model answer: unsupervised learning is most appropriate. The data is unlabelled — there are no pre-defined viewer categories to learn from — and the goal is to discover natural groupings of similar viewers, which is exactly what clustering, an unsupervised technique, does. Supervised learning could not be used because there are no labels to learn from, and the company is seeking to find unknown structure rather than predict a known answer. Identifying unsupervised learning and justifying it by the absence of labels and the discover-groupings goal is the mark.

QUESTION: Question five, the harder one. A bank has only two hundred transactions confirmed as fraudulent, but millions of unlabelled transactions, and labelling more requires expensive manual investigation. Recommend a training model and justify it, noting one risk. Pause, then play.

NARRATOR: Model answer: semi-supervised learning is the most appropriate choice. The bank has a small labelled set — two hundred confirmed fraud cases — and a very large unlabelled set of millions of transactions, and labelling more is expensive, which is precisely the situation semi-supervised learning is designed for. It uses the small labelled set to anchor the model and the large unlabelled set to learn the broader structure of normal versus suspicious behaviour, achieving better detection than the two hundred labelled cases alone could support, without the prohibitive cost of investigating millions of transactions. One risk: because so few labels are available and fraud is rare, the model may learn from biased or unrepresentative labels, or propagate incorrect pseudo-labels from the unlabelled data — so its predictions should be reviewed, since as with any model trained on human-derived labels, "garbage labels in, bias out". Naming semi-supervised, justifying it by the few-expensive-labels-plus-large-unlabelled situation, and identifying a real risk is the mark.

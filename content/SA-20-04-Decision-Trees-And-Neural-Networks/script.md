NARRATOR: The framing hook for the whole episode, the contrast in one line: a Tree is readable IF-questions; a Net is weighted neurons. The decision tree is a model you can read — literally a flowchart of yes-or-no questions, where you can trace exactly why it decided what it decided. The neural network is a model you mostly can't read — a web of weighted connections where the reasoning is spread across thousands of numbers nobody set by hand. Tree, readable; Net, opaque. NARRATOR: Start with the decision tree. A decision tree is a flowchart-like model that makes a prediction by asking a series of test questions about the input, following the branch that matches each answer, until it reaches a final prediction. It's a tree of if-then questions. As a programmer you've written this a hundred times: a decision tree is essentially a structured chain of nested if-statements — the same selection and branching logic you met in Year 11 algorithms. The difference in machine learning is that the questions and their order aren't hand-written by you; they're learned from data. The training process figures out which questions, asked in which order, best separate the outcomes. So it's learned branching logic.

NARRATOR: Now the parts, because naming them is a recall mark, and they have a memory hook. A tree has three kinds of node. The Root is the first question at the top — the single starting point. An internal Node is each further question along the way — a test on one feature. A Leaf is an end point — the final prediction, with no questions below it. Root, Node, Leaf — take the first letters, R-N-L. Top to bottom: Root at the top, Nodes in the middle, Leaves at the bottom. R-N-L is the tree from its root down to its leaves. The branches are the answers connecting one question to the next. R-N-L.

NARRATOR: Let me trace one, because "trace this decision tree for the given input" is an exam question with a stimulus, and you trace it exactly like running code in your head. Picture an email-priority tree. The Root: does the email contain the word "urgent"? If no, follow the "no" branch to a Leaf — medium priority, done. If yes, the next Node: does it contain "meeting"? If yes, Leaf — high priority. If no, next Node: is it from a known contact? If yes, Leaf — medium priority; if no, Leaf — low priority. Now trace a real input: an email that contains "urgent", does not contain "meeting", and is from a known contact. Root — contains "urgent"? Yes, go right. Next Node — contains "meeting"? No, go to the contact question. Next Node — from a known contact? Yes — Leaf, medium priority. The prediction is medium priority, and you can state the exact path that produced it. That traceability — naming every question that led to the answer — is the decision tree's superpower, and the whole basis of the comparison coming up. NARRATOR: Now the neural network — the other end of the spectrum. A neural network is a model loosely inspired by how brain neurons connect, built from many simple units called neurons, arranged in layers, with weighted connections between them. Build it bottom-up, because that's how the exam asks you to describe its structure. Start with one neuron. A single neuron does two things. First, it takes its inputs, multiplies each by a weight, and adds them up — that's a weighted sum, and the weights say how much each input matters. Second, it passes that sum through an activation function — a function that decides the neuron's output, often squashing it into a range or deciding whether the neuron fires. So a neuron is weighted sum, then activation. That's the whole unit. A two-input neuron is in Listing three.

NARRATOR: Now scale up. A neural network arranges these neurons in layers: an input layer that takes the raw features, one or more hidden layers in the middle that do the real work, and an output layer that produces the prediction. Every connection carries a weight, and learning means adjusting all those weights so the network's outputs get closer to the right answers — the how of that training is two episodes away, so for now hold: a network is layers of neurons connected by weights, and it learns by tuning the weights. A handy line: layers learn features — early layers pick up simple patterns like edges in an image, later layers combine them into complex ones like faces. That's why neural networks are the flagship for image recognition — the link straight back to last episode.

NARRATOR: Now the crucial contrast — and don't fall for the brain myth. A neural network is not a brain. It's loosely inspired by neurons, but it's just layers of weighted sums and activation functions — maths, not thought. Saying "it works like the human brain" is a band-four answer markers dislike; the accurate description is "a network of simple units that each compute a weighted sum and an activation, connected by learned weights". Use the accurate one.

NARRATOR: Here's the comparison the whole episode builds to — decision trees versus neural networks for explainability — because "compare these two models for explainability" is a four-mark question waiting for you. A decision tree is interpretable, or transparent: because it's a chain of readable questions, you can trace and explain exactly why it made any prediction — you point to the path. A neural network is a black box: its decision is spread across thousands or millions of weights with no human-readable rule, so even though it can be far more powerful on complex tasks like images and language, you usually can't say in plain terms why it produced a particular output. The trade-off: trees are more explainable but generally less powerful on complex patterns; networks are more powerful but far less explainable. Tree, readable; Net, opaque, powerful.

NARRATOR: This is where the threads from the security module reach across, which makes the comparison worth real marks. Why does explainability matter? Because of accountability — the last A of C-I-A-A-A — and because of bias and fairness, the ethical issue this module keeps flagging. If a neural network denies someone a loan or flags them as high-risk and nobody can explain why, you can't hold it accountable, you can't audit it for bias, and you can't tell the person why they were rejected. A transparent decision tree lets you trace the reason and check it for unfairness. So the explainability trade-off isn't technical trivia — it's the engineering reason that, in high-stakes or regulated decisions, an interpretable model is often preferred even when a black-box one scores slightly higher accuracy. Write that connection and you've lifted the answer into the top band.

NARRATOR: Worked example — the comparison done properly. "Compare decision trees and neural networks in terms of explainability." Weak answer: "Decision trees are simple and neural networks are complicated." That names nothing useful — no mechanism, no consequence. Strong answer: "A decision tree is highly explainable because it makes decisions through a traceable sequence of if-then questions, so a developer can follow the exact path from root to leaf and state precisely why a prediction was made. A neural network is a black box: its decision emerges from many learned weights across multiple layers, with no human-readable rule, so although it can model far more complex patterns, it is very difficult to explain why it produced a given output. The trade-off is that decision trees offer transparency and accountability at the cost of power on complex tasks, while neural networks offer power at the cost of interpretability — which is why a transparent model is often preferred for high-stakes decisions where the reasoning must be auditable." Hear the difference — the strong answer names the mechanism of each, states the trade-off, and ties it to accountability. Mechanism, trade-off, consequence.

NARRATOR: The other trap, beyond the brain myth, because it costs marks on both models: overfitting. A decision tree allowed to grow too deep stops learning the general pattern and starts memorising the exact training examples — it gets every training case right, then fails on new data. The same risk exists for an over-large neural network. So neither model is automatically better, and "bigger or deeper is better" is wrong — a too-complex model overfits. We meet overfitting properly in the regression episode; for now, a deep tree memorising data is the classic example, and the fix is to limit depth or prune the tree.

QUESTION: Question three. Describe the structure of a neural network. Four marks. Pause, then play.

NARRATOR: Model answer: a neural network is made of many simple units called neurons, arranged in layers — an input layer that receives the features, one or more hidden layers that process them, and an output layer that produces the prediction. Each neuron computes a weighted sum of its inputs and then applies an activation function to produce its output. Neurons are joined by weighted connections, and the network learns by adjusting those weights so its outputs better match the correct answers. Naming the layered structure, the neuron as weighted-sum-plus-activation, and the learned weights is the mark.

QUESTION: Question four, the comparison. Compare decision trees and neural networks in terms of explainability. Four marks. Pause, plan, then play.

NARRATOR: Model answer: a decision tree is highly explainable, because it reaches a prediction through a traceable sequence of if-then questions, so a developer can follow the path from root to leaf and state exactly why a decision was made. A neural network is a black box: its decision is distributed across many learned weights over multiple layers with no human-readable rule, so although it can model far more complex patterns, it is very difficult to explain a specific output. The trade-off is transparency versus power — decision trees favour interpretability and accountability, neural networks favour predictive power on complex data. Naming the mechanism behind each model's explainability and stating the trade-off earns full marks.

QUESTION: Question five, the harder one. A bank must choose between a decision tree and a neural network for approving loans, where regulators require every decision to be explainable. Recommend a model and justify it. Pause, weigh it, then play.

NARRATOR: Model answer: the decision tree is the more appropriate choice given the requirement that every decision be explainable. A decision tree produces a traceable path of if-then questions, so the bank can state precisely why any applicant was approved or declined, satisfying the regulator and allowing the model to be audited for bias and unfairness. A neural network, while potentially more accurate, is a black box whose decisions cannot be explained in human-readable terms, which would breach the explainability requirement and make it impossible to demonstrate the decision was fair or to give the applicant a reason. Therefore the bank should prefer the interpretable decision tree, accepting a possible small loss of predictive power in exchange for transparency and accountability. Recommending the tree and justifying it through explainability, accountability, and the ability to check for bias is the mark.

NARRATOR: Recap first, then we build the model everyone's heard of. Last episode was regression and the core algorithm types, hook L-L-K plus polynomial. Linear regression predicts a number by fitting a line. Logistic regression — despite its name — classifies, predicting yes-or-no via the sigmoid, the S-shaped function that maps any number to a probability between zero and one. Polynomial regression fits a curve, and is how you overfit. K-nearest neighbour predicts from the K most similar examples. Models are objects with fit and predict — fit learns the parameters, predict applies them. And the two ideas that matter most: overfitting, fitting the training data so closely you memorise noise and fail on new data; and the train-test split, evaluating on unseen data, never on training data, which is how you catch overfitting. Remember Zillow, the overfitted price model that lost hundreds of millions.

NARRATOR: Start with the perceptron, the simplest possible neural network — a single neuron. You already have the pieces from last chapter, so this is assembly, not new parts. A perceptron takes several inputs, each multiplied by a weight; it adds them all up, plus a bias — that's the weighted sum; and it passes the sum through an activation function to produce its output. Inputs, weights, bias, weighted sum, activation, output. The weights are the learned parameters — they say how much each input matters — and the bias is an adjustable threshold. The classic teaching example is making a perceptron learn a logic gate: given two binary inputs, output one if both are one for AND, or one if either is one for OR. The perceptron learns the weights and bias that produce the right output. The perceptron class is in Listing one. One line: a perceptron is a single neuron that computes a weighted sum of its inputs plus a bias, then applies an activation function to produce an output.

NARRATOR: Now the forward pass — the step that turns inputs into a prediction, and what "trace this" questions ask about. The forward pass is running the input through the network: multiply each input by its weight, sum them with the bias, apply the activation, and for a multilayer network, feed that output forward as the input to the next layer, all the way to the output. It's called "forward" because data flows forward, input to output. Let me trace a tiny one. A perceptron has two inputs, both with weight zero-point-five, a bias of minus-zero-point-seven, and a step activation that outputs one if the sum is at least zero, else zero. Feed in inputs one and one: the weighted sum is zero-point-five plus zero-point-five minus zero-point-seven, equals zero-point-three — at least zero, so output one. Feed in one and zero: zero-point-five minus zero-point-seven equals minus-zero-point-two — below zero, so output zero. That perceptron has learned the AND gate: it only fires when both inputs are one. That's a forward pass, traced by hand — multiply, sum, activate. The pseudocode is in Listing three.

NARRATOR: Now scale to a multilayer network — a multilayer perceptron — and describe its structure properly, because "describe the structure of a neural network" is a four-mark question. The structure is three kinds of layer. An input layer receives the raw features — one input neuron per feature. One or more hidden layers in the middle, where each neuron takes the previous layer's outputs, computes its own weighted sum and activation, and passes it forward. And an output layer that produces the final prediction. Every connection between neurons carries its own weight, so a network has many weights — that's where its knowledge lives. And here's the key line, the reason you bother with hidden layers: layers learn features. Early layers detect simple patterns — in an image, edges and corners — and later layers combine those into complex ones — shapes, then faces. Stacking layers lets a network model relationships a single perceptron or a straight line never could. One line: a neural network is an input layer, one or more hidden layers, and an output layer, with neurons connected by weighted links, where each neuron computes a weighted sum and an activation, and the layers progressively learn more complex features.

NARRATOR: Now the heart of today — how a neural network learns from data. This is where students hand-wave "it just figures it out", and the marker wants the actual loop. Here's the memory hook: P-L-A — Predict, Loss, Adjust — repeat. Three steps, over and over. Predict: run a forward pass on a training example to get the network's output. Loss: compare that output to the known correct answer — the label — and measure how wrong it is; that error measure is the loss. Adjust: nudge the weights a little in the direction that reduces the error, so next time the prediction is closer. Then repeat across the whole dataset, many times. Each full pass over the training data is an epoch, and over many epochs the weights settle into values that make good predictions. That's it. Predict, Loss, Adjust. Say it out loud: the network learns by repeatedly predicting, measuring how wrong it was, and adjusting its weights to be less wrong.

NARRATOR: Two terms to attach so your answer sounds expert, but don't get lost in the maths. The "figure out which way to nudge each weight" step is, technically, backpropagation — the algorithm that pushes the error backwards through the layers to work out how much each weight contributed to the mistake. And the "nudge in the direction that reduces error" is gradient descent — stepping downhill on the error. The examinable intuition is still the plain version: the network compares its prediction to the right answer and adjusts its weights to reduce the error, over and over. Predict, Loss, Adjust. Name backpropagation and gradient descent for polish, but the loop is the substance. The training loop in pseudocode is in Listing two.

NARRATOR: And tie it back, because this loop is exactly what we glimpsed in regression. When linear regression nudged its slope and intercept to reduce error over many epochs, that was the same idea — predict, measure error, adjust parameters. A neural network does that for thousands of weights across layers instead of two parameters. So you already met P-L-A in miniature last episode; a network just scales it up. Worth saying in an answer: training a neural network is the same predict-measure-adjust optimisation as fitting a regression, applied to many weighted-sum units at once.

NARRATOR: Now the traps, both of which cost marks. First, the hand-wave. "The network just learns the patterns" is a band-four non-answer. The marker wants the mechanism: it predicts, measures the error against the known answer, and adjusts its weights to reduce that error, repeatedly. Always give the loop. Second, muddling the vocabulary. Keep these straight: a neuron is a single unit that computes a weighted sum and activation; a layer is a group of neurons at the same stage; a weight is the learned number on a connection; and the bias is the per-neuron adjustable threshold. Neuron, layer, weight, bias — don't blur them. A common error is calling the whole network "a neuron", or calling weights "neurons". Precision here is straightforward marks.

NARRATOR: And carry forward the overfitting warning, because it applies to networks too. A network with too many layers and weights can overfit just like a high-degree polynomial — it memorises the training data and fails on new data — so the train-test split from last episode still applies: evaluate on unseen data. A network's power is also its danger: more capacity means more ability to overfit. So "bigger network is better" is wrong for the same reason "higher-degree polynomial is better" was wrong. Same lesson, bigger model.

NARRATOR: Worked example — "explain how a neural network learns from data", done properly. Weak answer: "It processes the data through its layers and learns the patterns automatically." That's the hand-wave — no mechanism, no loop. Strong answer: "A neural network learns through repeated cycles of predict, measure, adjust. For each training example it performs a forward pass — passing the inputs through the weighted connections and activation functions of each layer to produce a prediction. It then calculates the loss — how far the prediction is from the known correct answer. Using backpropagation, it determines how much each weight contributed to the error, and using gradient descent it adjusts every weight slightly to reduce the loss. Repeating this over many examples and many epochs gradually tunes the weights so the network makes accurate predictions. To confirm it has learned the general pattern rather than memorising the data, it is evaluated on a separate test set." Hear the difference — the strong answer names the forward pass, the loss, the weight adjustment, the repetition, and the generalisation check. Predict, loss, adjust, repeat, validate.

QUESTION: Question three. Describe the structure of a neural network. Four marks. Pause, then play.

NARRATOR: Model answer: a neural network consists of layers of neurons. The input layer receives the raw features, one neuron per feature; one or more hidden layers in the middle each take the previous layer's outputs, compute a weighted sum and an activation, and pass the result forward; and the output layer produces the final prediction. Neurons are connected by weighted links, and these weights — together with the biases — are the learned parameters that hold the network's knowledge. The hidden layers progressively learn more complex features, from simple patterns to combinations of them. Naming the three layer types, the weighted connections, and the per-neuron weighted-sum-and-activation earns full marks.

QUESTION: Question four, the big one. Explain how a neural network learns from data. Six marks. Pause, plan, then play.

NARRATOR: Model answer: a neural network learns through repeated cycles of predicting, measuring error, and adjusting its weights. For each training example it performs a forward pass, passing the inputs through the weighted connections and activation functions of each layer to produce a prediction. It then calculates the loss — a measure of how far the prediction is from the known correct label. Using backpropagation it works out how much each weight contributed to the error, and using gradient descent it adjusts every weight slightly in the direction that reduces the loss. This predict-loss-adjust cycle is repeated across all training examples for many epochs, gradually tuning the weights so predictions become accurate. To confirm it has learned the general pattern rather than memorising the training data, its performance is checked on a separate, unseen test set. A full-mark answer names the forward pass, the loss, the weight adjustment via backpropagation and gradient descent, the repetition over epochs, and the validation on unseen data.

QUESTION: Question five, the harder one. A team builds a very large neural network that achieves perfect accuracy on its training data but performs poorly in real use. Diagnose the problem and recommend two actions. Pause, then play.

NARRATOR: Model answer: the network has overfitted. Perfect training accuracy with poor real-world performance indicates it has memorised the specific training examples, including their noise, rather than learning the general pattern, so it fails to generalise to new data — a greater risk for a very large network, because its many weights give it the capacity to memorise. Two recommended actions: first, simplify the network — reduce the number of layers or neurons, or apply regularisation — so it cannot contort itself to the training data; and second, evaluate it on a separate held-out test set during development and gather more representative training data, so overfitting is detected and corrected before deployment. Diagnosing overfitting from the train-versus-real gap and giving two genuine corrective actions is the mark.
